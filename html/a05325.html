Installation instructions

Clone the GridTools repo at a specific revision

cd ${HOME}
git clone https://github.com/GridTools/gridtools.git
cd gridtools
git checkout galerkin4gt_010

Install gridtools

You need the Boost third party library

mkdir build
cd build
cmake \
-DCMAKE_INSTALL_PREFIX:PATH=${HOME}/local/GridTools \
-DCMAKE_BUILD_TYPE=RELEASE \
-DUSE_GPU:BOOL=ON \
-DCUDA_ARCH:STRING="sm_35" \
-DUSE_MPI:BOOL=OFF \
-DSINGLE_PRECISION:BOOL=OFF \
-DENABLE_CXX11:BOOL=ON \
-DENABLE_PYTHON:BOOL=OFF \
-DENABLE_PERFORMANCE_METERS:BOOL=ON \
-DSTRUCTURED_GRIDS=ON \
..
make
make install

In this example we are installing GridTools without MPI support, with GPU, double precision and on structured grids

Available options are summarized in the following table:

Optional: Small tutorial on how to install Xdmf

This part is optional. Required third party libraries for Xdmf are Xml2 and HDF5. HDF5 is a general purpose parallel binary file format. Xdmf is an XML format to describe the output results (which may be stored in binary format, e.g. HDF5).

The HDF5 API has undergone some changes, and they were not always reflected in the Xdmf API. The current instructions refer to the version 1.10 for HDF5, and the last version from Xdmf2 Git repository (i.e. version 2.0 for Xdmf).

You can either compile or install the HDF5 library directly. To compile it go to the source directory and run the command

mkdir build
cd build
cmake -DCMAKE_INSTALL_PREFIX=${HOME}/local/hdf5 ..
make install

For simplicity we show how to install HDF5 without the support for parallel I/O.

One can clone Xdmf from the Git repo:

git clone git://xdmf.org/Xdmf.git

At the moment of writing there's a mismatch in HDF5's last version CMake module and the Xdmf CMake script. We are considering hdf5-1.10.0-patch1, and a snapshot of Xdmf corresponding to the following hash 009e6e18b82ba81e1d9fd0bc9a454269b0d24987 (Fri Feb 10 2017). There is a very simple patch which fixes the two versions: open the file

Xdmf/core/CMakeLists.txt

replace the lines

find_package(HDF5 REQUIRED)
if(HDF5_FOUND)

with

find_package(HDF5 REQUIRED HINTS ${HDF5_ROOT}/share/cmake)
if(HDF5_FOUND)
set(HDF5_INCLUDE_DIRS ${HDF5_INCLUDE_DIR})

so that the correct configuration for HDF5 is find, and add the right include directories are used (the problem is that the HDF5Config.cmake file shipped with this version of HDF5 defines a CMake variable called HDF5_INCLUDE_DIR, while the Xdmf library expects a variable called HDF5_INCLUDE_DIRS, with an extra "S"). Make sure that the environment variable HDF5_ROOT is defined, and set to the root of the HDF5 installation (you need both the HDF5 library and the headers).

Now we are ready to compile and install Xdmf:

cd Xdmf
mkdir build
cd build

configure using CMake (note that you need also libxml2 as a dependency, and that we disable python in this example)

cmake -DCMAKE_INSTALL_PREFIX=~/local/xdmf -DBUILD_SHARED_LIBS=1 -DXDMF_WRAP_PYTHON=0 -DHDF5_ROOT=~/local/hdf5 -Wno-dev ..
make
make install

This installs Xdmf without python support, and with the HDF5 from our local path.

Install Intrepid (Trilinos)

Some packages (Intrepid, Teuchos and Kokkos) are required by galerking4gt. We take from Trilinos the value of the basis functions in the cubature points the definition of element cells and their boundary some tools to deal with these element-local quantities

These features are simple but tedious, and we do not want to replicate them in the library.

To install the aforementioned Trilinos packages download Trilinos from the public git repo

git clone https://software.sandia.gov/trilinos/repositories/publicTrilinos

then create a build folder and build/install only the required packages

mkdir build
cd build
cmake \
-DCMAKE_INSTALL_PREFIX:PATH=~/local/publicTrilinos \
-DTrilinos_ENABLE_ALL_OPTIONAL_PACKAGES:BOOL=OFF \
-DTrilinos_ENABLE_Intrepid:BOOL=ON \
-DTrilinos_ENABLE_Kokkos:BOOL=ON \
-DTrilinos_ENABLE_CXX11=ON \
../
make
make install

Install Galerkin4GT

We proceed with the compilation of the examples in the galerkin4gt repo:

cd ${HOME}
git clone https://github.com/eth-cscs/galerkin4gt.git
cd galerkin4gt

For the configuration we need some libraries from Trilinos installed, namely Intrepid, Teuchos and Kokkos, and the blas and lapack libraries. There is also an optional Xdmf dependency for i/o.

cmake \
-DGRIDTOOLS_ROOT:PATH=${HOME}/local/GridTools \
-DCMAKE_BUILD_TYPE=RELEASE \
-DENABLE_NVCC_OUTPUT:BOOL=OFF \
-DENABLE_XDMF:BOOL=OFF \
-DENABLE_INTREPID:BOOL=ON \
-DINTREPID_SOURCE_DIR:PATH=~/local/publicTrilinos/include \
-DINTREPID_LIB_DIR:PATH=~/local/publicTrilinos/lib \
-DBLAS_DIR:PATH=/usr/lib \
-DLAPACK_DIR:PATH=/usr/lib \
~/src/GridTools/galerkin4gt

compile:

make

Note if the CMake variable ENABLE_NVCC_OUTPUT is set to false (the default value) all the messages NVCC prints are not displayed. The reason for such flag is that nvcc prints tons of nonrelevant warnings which cannot be filtered out by compiler flags. If you want to see the actual errors (together with the tons of warnings) set the ENABLE_NVCC_OUTPUT flag to ON, and recompile only the target which is failing.

Available options are summarized in the following table:

Syntax

ENABLE_NVCC_OUTPUT if set to OFF redirects to /dev/null the output of nvcc OFF
ENABLE_XDMF looks for Xdmf library, to dave output files in hdf5 format OFF

XDMF_LIBS (XDMF_INCLUDE_DIRS) set this variable to the path of the Xdmf libs (include dirs) when it is not found automatically

OFF

LAPACK_HARDCODED_LIB same as above, but for lapack N/A
