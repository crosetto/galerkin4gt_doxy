DG Flux Implementation Example

In this example we demonstrate how to compute an integral on a boundary face

The definitions section is composed by a first part, which is similar to the one in the previous examples

132  using matrix_storage_info_t=storage_info< __COUNTER__, layout_tt<6>>;
133  using matrix_type=storage_t< matrix_storage_info_t >;
134 
135  using fe=reference_element<1, Lagrange, Hexa>;
136  using geo_map=reference_element<1, Lagrange, Hexa>;
137  using cub=cubature<fe::order(), fe::shape()>;
138  using geo_t = intrepid::geometry<geo_map, cub>;
139  geo_t geo;

Notice however that we did not define the discretization an geometric map, since in this example we will be integrating only on the boundary. A second part is necessary to define the cubature on the boundary faces, and the discretization:

143  using bd_cub_t = intrepid::boundary_cub<geo_map, cub::cubDegree>;
144  using bd_discr_t = intrepid::boundary_discr<bd_cub_t, 1>;
145  bd_cub_t bd_cub_;
146  bd_discr_t bd_discr_(bd_cub_, 1);
147 
148  bd_discr_.compute(Intrepid::OPERATOR_GRAD);

We compute the gradient on the faces of the reference finite element, evaluated in the quadrature points on the boundary NOTE the

bd_discr_

instantiation is given an ordinal (1 in this case). This ordinal identify a face in the exahedron, and can be substituted by a list of ordinals

bd_discr_(bd_cub_,0,1,2,3,4,5)

if we consider all faces. So in this case we want to compute the boundary integral on face number 1 of each element. The face ordinals are assigned with the following convention:

index normals .____.
0 (0,-1,0) / 0 /|
1 (1,0,0) .____. |5
2 (0,1,0) | |1. z
3 (-1,0,0) 3| 4 |/ x__/
4 (0,0,-1) .____. |
5 (0,0,1) 2 y

We define for this case only the assembly_base and a bd_assembly, since we do not need the assembler for the volumetric part

153  using as_base=assembly_base<geo_t>;
154  using as=bd_assembly<bd_discr_t>;
155  as_base assembler_base(d1,d2,d3);
156  as assembler(bd_discr_,d1,d2,d3);

We instantiate the matrix which will contain the mass matrix of the right-boundary of each element

174  matrix_storage_info_t meta_(d1,d2,d3,fe::basis_cardinality(),fe::basis_cardinality(), 1/*faces*/);
175  matrix_type mass_(meta_, 0., "mass");

Notice that this matrix is represented with a 6D array, three space dimensions identifying the element, two dimensions for specifying the degrees of freedom (row/column of the local matrix), and one extra dimension identifying the face (useless in this case since we only consider one face).

We define in a similar way the vectors used in the computation

179  using vector_storage_info_t=storage_info< __COUNTER__, layout_tt<5> >;
180  using vector_type=storage_t< vector_storage_info_t >;
181  vector_storage_info_t vec_meta_(d1,d2,d3,fe::basis_cardinality(), 1/*face*/);
182  vector_type u_(vec_meta_, 2., "u");//initial solution
183  vector_type flux_(vec_meta_, 0., "flux");
184  vector_type integrated_flux_(vec_meta_, 0., "integrated flux");

Then as usual we define a placeholder for each argument we want to pass to the computation

the aggregator

208 
209  // appending the placeholders to the list of placeholders already in place
210  auto domain=gt::aggregator_type<arg_list>(
211  (p_bd_jac()=assembler.bd_jac())
212  , (p_normals()=assembler.normals())
213  , (p_bd_measure()=assembler.bd_measure())
214  , (p_bd_weights()=assembler.bd_backend().bd_cub_weights())
215  , (p_ref_normals()=assembler.ref_normals())
216  , (p_bd_phi()=assembler.bd_backend().val())
217  , (p_bd_dphi()=assembler.bd_backend().grad())
218  , (p_grid_points()=assembler_base.grid())
219  , (p_mass()=mass_)
220  , (p_bd_discr_dphi()=bd_discr_.grad())
221  , (p_bd_discr_phi()=bd_discr_.val())
222  , (p_u()=u_)
223  , (p_flux()=flux_)
224  , (p_integrated_flux()=integrated_flux_)
225  );

and the computation

234  auto computation=gt::make_computation< BACKEND >(
235  domain, coords,
236  make_multistage
237  (
238  execute<forward>()
239  // evaluate the cell Jacobian matrix on the boundary (given the basis functions derivatives in those points)
240  , gt::make_stage<functors::update_bd_jac<bd_discr_t , enumtype::Hexa> >(p_grid_points(), p_bd_dphi(), p_bd_jac())
241  // compute the normals on the quad points from the jacobian above, transporting the normals from the reference configuration, unnecessary. (can be computed also as the cross product of the first 2 columns of J)
242  , gt::make_stage<functors::compute_face_normals<bd_discr_t> >(p_bd_jac(), p_ref_normals(), p_normals())
243  // surface integral:
244  // compute the measure for the surface integral
245  // | / d(phi_x)/du d(phi_x)/dv 1 \ |
246  // det(J) = | | d(phi_y)/du d(phi_y)/dv 1 | |
247  // | \ d(phi_z)/du d(phi_z)/dv 1 / |
248  , gt::make_stage<functors::measure<bd_discr_t, 1> >(p_bd_jac(), p_bd_measure())
249  // evaluate the mass matrix
250  , gt::make_stage<integration<fe, bd_cub_t::bd_cub> >(p_bd_measure(), p_bd_weights(), p_bd_discr_phi(), p_bd_discr_phi(), p_mass()) //mass
251  // compute the flux, this line defines the type of approximation we choose for DG
252  // (e.g. upwind, centered, Lax-Wendroff and so on)
253  // NOTE: if not linear we cannot implement it as a matrix
254  , gt::make_stage<functors::bassi_rebay<bd_discr_t> >(p_u(), p_u(), p_flux())
255  // integrate the flux: mass*flux
256  , gt::make_stage< functors::matvec_BdxBdxBd >( p_flux(), p_mass(), p_integrated_flux() )
257  ));
258 
259  computation->ready();
260  computation->steady();
261  computation->run();
262  computation->finalize();

The different steps of the computation are:

  • compute the jacobian matrix on the boundary
  • compute the face normals
  • compute the boundary measure (i.e. the norm of the cross product of the tangent derivatives )
  • evaluate the mass matrix
  • evaluate a flux across the boundary
  • integrate the flux on the face (i.e. perform a multiplication with the boundary mass matrix)

namespace gdl{
/* compute the integral on the boundary of a field times the normals
note that we use here the traces of the basis functions, i.e. the basis functions
evaluated on the quadrature points of the boundary faces.
*/
using namespace gt::expressions;
template <typename FE, typename BoundaryCubature>
struct integration {
using fe=FE;
using bd_cub=BoundaryCubature;
using jac_det=gt::accessor< 0, enumtype::in, gt::extent<0,0,0,0>, 5 >;
using weights=gt::accessor< 1, enumtype::in, gt::extent<0,0,0,0>, 4 >;
using phi_trace=gt::accessor< 2, enumtype::in, gt::extent<0,0,0,0>, 3 >;
using psi_trace=gt::accessor< 3, enumtype::in, gt::extent<0,0,0,0>, 3 >;
using out=gt::accessor< 4, enumtype::inout, gt::extent<0,0,0,0>, 6 >;
using arg_list=boost::mpl::vector<jac_det, weights, phi_trace, psi_trace, out> ;
template <typename Evaluation>
GT_FUNCTION
static void Do(Evaluation const & eval, x_interval) {
gt::dimension<4> quad;
gt::dimension<4> dofI;
gt::dimension<5> dofJ;
uint_t const num_cub_points=eval.template get_storage_dim<3>(jac_det());
uint_t const basis_cardinality = eval.template get_storage_dim<3>(phi_trace());
//NOTE: missing the loop on the faces for this example
//loop on the basis functions (interpolation in the quadrature point)
for(short_t P_i=0; P_i<basis_cardinality; ++P_i) // current dof
{
for(short_t P_j=0; P_j<basis_cardinality; ++P_j) // current dof
{
float_type partial_sum=0.;
for(ushort_t q_=0; q_<num_cub_points; ++q_){
partial_sum += eval(!phi_trace(P_i,q_)*!psi_trace(P_j, q_)*jac_det(quad+q_));
}
eval(out(dofI+P_i, dofJ+P_j))=partial_sum;
}
}
}
};
}//namespace gdl
int main(int argc, char ** argv){
using namespace gdl;
using namespace enumtype;
uint_t d1;
uint_t d2;
uint_t d3;
if(argc==1){
d1 = 5;
d2 = 5;
d3 = 5;
}
else{
if(argc!=4) {
printf("usage: \n >> stiffness <d1> <d2> <d3>\n");
exit(-666);
}
//dimensions of the problem (in number of elements per dimension)
d1= atoi(argv[1]);
d2= atoi(argv[2]);
d3= atoi(argv[3]);
}
//defining the assembler, based on the Intrepid definitions for the numerics
using matrix_storage_info_t=storage_info< __COUNTER__, layout_tt<6>>;
using matrix_type=storage_t< matrix_storage_info_t >;
using fe=reference_element<1, Lagrange, Hexa>;
using geo_map=reference_element<1, Lagrange, Hexa>;
using cub=cubature<fe::order(), fe::shape()>;
using geo_t = intrepid::geometry<geo_map, cub>;
geo_t geo;
using bd_cub_t = intrepid::boundary_cub<geo_map, cub::cubDegree>;
using bd_discr_t = intrepid::boundary_discr<bd_cub_t, 1>;
bd_cub_t bd_cub_;
bd_discr_t bd_discr_(bd_cub_, 1);
bd_discr_.compute(Intrepid::OPERATOR_GRAD);
using as_base=assembly_base<geo_t>;
using as=bd_assembly<bd_discr_t>;
as_base assembler_base(d1,d2,d3);
as assembler(bd_discr_,d1,d2,d3);
//constructing a structured cartesian grid
for (uint_t i=0; i<d1; i++)
for (uint_t j=0; j<d2; j++)
for (uint_t k=0; k<d3; k++)
for (uint_t point=0; point<fe::basis_cardinality(); point++)
{
assembler_base.grid()( i, j, k, point, 0)= (i + (1+geo.grid()(point, 0, 0))/2.)/d1;
assembler_base.grid()( i, j, k, point, 1)= (j + (1+geo.grid()(point, 1, 0))/2.)/d2;
assembler_base.grid()( i, j, k, point, 2)= (k + (1+geo.grid()(point, 2, 0))/2.)/d3;
}
//defining the mass matrix: d1xd2xd3 elements
matrix_storage_info_t meta_(d1,d2,d3,fe::basis_cardinality(),fe::basis_cardinality(), 1/*faces*/);
matrix_type mass_(meta_, 0., "mass");
using vector_storage_info_t=storage_info< __COUNTER__, layout_tt<5> >;
using vector_type=storage_t< vector_storage_info_t >;
vector_storage_info_t vec_meta_(d1,d2,d3,fe::basis_cardinality(), 1/*face*/);
vector_type u_(vec_meta_, 2., "u");//initial solution
vector_type flux_(vec_meta_, 0., "flux");
vector_type integrated_flux_(vec_meta_, 0., "integrated flux");

typedef gt::arg<1, typename as::face_normals_type > p_normals;
typedef gt::arg<2, typename as::storage_type > p_bd_measure;
typedef gt::arg<3, typename as::boundary_t::weights_storage_t> p_bd_weights;
typedef gt::arg<4, typename as::boundary_t::tangent_storage_t> p_ref_normals;
typedef gt::arg<5, typename as::boundary_t::basis_function_storage_t> p_bd_phi;
typedef gt::arg<6, typename as::boundary_t::grad_storage_t> p_bd_dphi;
typedef gt::arg<7, typename as_base::grid_type > p_grid_points;
// defining the placeholder for the mass
typedef gt::arg<8, matrix_type> p_mass;
// defining the placeholder for the local gradient of the element boundary face
typedef gt::arg<9, bd_discr_t::grad_storage_t> p_bd_discr_dphi;
typedef gt::arg<10, bd_discr_t::basis_function_storage_t> p_bd_discr_phi;
typedef gt::arg<11, vector_type> p_u;
typedef gt::arg<12, vector_type> p_flux;
typedef gt::arg<13, vector_type> p_integrated_flux;
typedef boost::mpl::vector<p_bd_jac, p_normals, p_bd_measure, p_bd_weights, p_ref_normals, p_bd_phi, p_bd_dphi, p_grid_points, p_mass, p_bd_discr_dphi, p_bd_discr_phi, p_u, p_flux, p_integrated_flux> arg_list;
// appending the placeholders to the list of placeholders already in place
auto domain=gt::aggregator_type<arg_list>(
(p_bd_jac()=assembler.bd_jac())
, (p_normals()=assembler.normals())
, (p_bd_measure()=assembler.bd_measure())
, (p_bd_weights()=assembler.bd_backend().bd_cub_weights())
, (p_ref_normals()=assembler.ref_normals())
, (p_bd_phi()=assembler.bd_backend().val())
, (p_bd_dphi()=assembler.bd_backend().grad())
, (p_grid_points()=assembler_base.grid())
, (p_mass()=mass_)
, (p_bd_discr_dphi()=bd_discr_.grad())
, (p_bd_discr_phi()=bd_discr_.val())
, (p_u()=u_)
, (p_flux()=flux_)
, (p_integrated_flux()=integrated_flux_)
);
auto coords=gt::grid<axis>({1, 0, 1, (uint_t)d1-1, (uint_t)d1},
{1, 0, 1, (uint_t)d2-1, (uint_t)d2});
coords.value_list[0] = 0;
coords.value_list[1] = d3-1;
auto computation=gt::make_computation< BACKEND >(
domain, coords,
make_multistage
(
execute<forward>()
// evaluate the cell Jacobian matrix on the boundary (given the basis functions derivatives in those points)
, gt::make_stage<functors::update_bd_jac<bd_discr_t , enumtype::Hexa> >(p_grid_points(), p_bd_dphi(), p_bd_jac())
// compute the normals on the quad points from the jacobian above, transporting the normals from the reference configuration, unnecessary. (can be computed also as the cross product of the first 2 columns of J)
, gt::make_stage<functors::compute_face_normals<bd_discr_t> >(p_bd_jac(), p_ref_normals(), p_normals())
// surface integral:
// compute the measure for the surface integral
// | / d(phi_x)/du d(phi_x)/dv 1 \ |
// det(J) = | | d(phi_y)/du d(phi_y)/dv 1 | |
// | \ d(phi_z)/du d(phi_z)/dv 1 / |
, gt::make_stage<functors::measure<bd_discr_t, 1> >(p_bd_jac(), p_bd_measure())
// evaluate the mass matrix
, gt::make_stage<integration<fe, bd_cub_t::bd_cub> >(p_bd_measure(), p_bd_weights(), p_bd_discr_phi(), p_bd_discr_phi(), p_mass()) //mass

// NOTE: if not linear we cannot implement it as a matrix
, gt::make_stage<functors::bassi_rebay<bd_discr_t> >(p_u(), p_u(), p_flux())
// integrate the flux: mass*flux
, gt::make_stage< functors::matvec_BdxBdxBd >( p_flux(), p_mass(), p_integrated_flux() )
));
computation->ready();
computation->steady();
computation->run();
computation->finalize();

}
