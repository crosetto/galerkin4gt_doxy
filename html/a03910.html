Detailed Description

template<ushort_t N_DOF0, ushort_t N_DOF1, ushort_t N_DOF2>
class gdl::functors::hexahedron_vector_assemble< N_DOF0, N_DOF1, N_DOF2 >

mesh vector (single dof indexed object) assemble functor

Template Parameters
Numberof single hexahedron dofs along x direction
Numberof single hexahedron dofs along y direction
Numberof single hexahedron dofs along z direction

hypotheses:

  • dof number is the same along each element direction (total number of dofs pre element is n_dofs^3)
  • reference frame axes are defined as follows

z / -x | y

  • dofs are ordered in the input matrix according to the following rule

    4-—5 / /| 0-—1 | | | 7 | |/ 2-—3

(not represented internal dofs follow the same rule)

  • each hexahedron is responsible for the assemble of a set of contributions related to the dof shared with the adjacent hexaedrons in negative x,y and z direction (each hexahedron takes contribution from 7 adjacent hexahedrons). Particularly, considering a single face the dofs are grouped as follows

F–B–B–B–G ---—x | | | C–A–A–A–E | | | y C–A–A–A–E | | C–A–A–A–E | | H–D–D–D–I

The assemble (gathering) is performed for heach hexahedron on the following dof groups :

Loop_number | Group | Number_contr

1 | A | 2 2 | B+C | 4

3 | F | 8

The same calculation is performed for the faces on xz and xy planes.

As it can be seen some contributions are missing (e.g. E): those elements are included in the calculations of the adjacent (in positive x/y/z direction) hexahedron.
