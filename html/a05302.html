Gather from neighbours

This example shows how to implement a step in Finite Volumes or DG discretizations, i.e. the exchange of values at the interface between neighbouring elements. This is exemplified by the following picture, representing a classical Von Neumann stencil:

Notice that for continuous finite elements this stencil would be part of the assembly step, but also the points inside the elements would be involved in the assembly, since the basis funcitons cannot be discontinuous across the elements. Furthermore in the case of continuous finite elements we cannot limit ourselves to a von Neumann stencil (i.e. neighbouring elements sharing an entire face), also the elements sharing a single point/edge can contribute to the assembled matrix.

We remark that the for ths example we make the following hypotheses:

  • dof number is the same along each element direction (total number of dofs pre element is n_dofs^3)
  • reference frame axes are defined as follows
    z
    /
    -- x
    |
    y
  • dofs are ordered in the input matrix according to the following rule
    4----5
    / /|
    0----1 |
    | | 7
    | |/
    2----3
    (not represented internal dofs follow the same rule)
